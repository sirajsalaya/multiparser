# multipartbody v1.0.0

## What it solves

`multipartbody` is an Express multipart middleware for projects that need files to live directly inside `req.body`, especially for nested arrays and object payloads.

It is designed for request shapes where Multer's `req.file` and `req.files` split is inconvenient.

## Highlights

- Parses `multipart/form-data` into `req.body`
- Supports nested key paths like `profile.avatar` and `docs[0].file`
- Supports repeated indexed record blocks without collapsing all values into one object
- Supports top-level array payloads
- Includes TypeScript types, Multer-style limits, MIME filtering, and pluggable storage

## Quick example

```ts
import express from 'express';
import multipartBody from 'multipartbody';

const app = express();

app.post('/upload', multipartBody(), (req, res) => {
  res.json(req.body);
});
```

## Example request

```bash
curl --location 'http://localhost:3000/upload' \
  --form '[0].id=1' \
  --form '[0].docs[0].file=@"/path/to/file-1.jpg"' \
  --form '[1].id=2' \
  --form '[1].docs[0].file=@"/path/to/file-2.jpg"'
```
